Build Blog on GitHub by Hexo

Posted on 2017-11-23 | In Build Blog

本文简要介绍了基于GitHub和Hexo在Mac上搭建个人博客的过程以及theme更改、域名设置、图片上传等常见问题提供了解决方案。首先，本文介绍了基于GitHub和Hexo搭建博客的三个关键步骤：借助hexo创建本地博客、在GitHub上创建repository以及通过SSH keys将前两者进行关联。然后，介绍了文章的创建和发布、theme的更改、域名的设置、本地图片插入等常见问题的解决方案。

1. 环境

在GitHub上搭建个人博客，除了需要一个GitHub账号外，本地环境需要必备如下几个应用程序：

  • Node.js：用于生成静态页面。
  • Git：可将本地的Hexo内容提交到GitHub上。
  • Hexo

4.4 本地图片

在博客中插入图片：

  • 方法一：借助某些工具或服务来生成图片的URL，插入文章中。例如

    • 借助微博图床
    • 借助七牛云

    • 借助V2ex上网友做的图床网站https://sm.ms

      等类似的方法来生成图片的URL。这种方法的优势在于图片的URL在任意文章中均可以使用，劣势在于文章图片较多时不方便管理、七牛云在使用一定量后是收费的。

  • 方法二：本地图片上传。借助hexo-asset-image插件来实现本地图片的上传和加载。具体过程如下：

    1. 打开博客根目录下的_config.yml文件，修改post_asset_folder字段的值为true。这样每次通过hexo new新建文章时，会在同级目录下创建相同名字的文件夹。例如新创建的文章的名字为new-artile-name.md，相应的文件夹的名字为new-article-name。

    2. 终端cd至博客根目录，执行

    3. 将准备插入文章的图片复制到与文章同名的文件夹，然后在文章中使用标准的图片插入方式插入即可。例如![文字](new-article-name/图片名.jpg)

4.5 代码托管

关于异地更新博客文章或更换电脑后对博客的更新等类似问题。

  • 方法一：备份所有的Blog文件和代码。该方法的缺点在于更新不及时，需要手动备份，比较麻烦。
  • 方法二：在GitHub新建仓库存放Blog的文件和代码。该方法会使博客的源代码公开，不是一个好办法。
  • 方法三：在Coding.net上，个人用户可以免费使用一定数量的私人项目，因此可以考虑将代码托管其中。这样可以解决私密性和托管的问题。

4.6 部署至Coding

为了方便国内的访问，也多一些选择可能，有些还会把博客同时部署到Coding.net，同时在域名解析中增加国内域名的解析。

本人目前没有在Coding上进行过部署，所以不做具体介绍。
